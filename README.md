# Visual Learn Everything

**Drop in any document вҖ” get back an interactive study session.**

Visual Learn Everything turns a **PDF, Markdown, or text file** into a rich,
interactive learning workspace: a zoomable **mind map**, a **summary**,
**keyвҖ‘concept cards**, flipвҖ‘able **flashcards**, an autoвҖ‘graded **quiz**, and a
gallery of **images extracted from the document** вҖ” all generated by a GenAI
model.

> Point it at a textbook chapter, a research paper, a set of lecture notes, or a
> README, and start learning in seconds.

---

## I. Features

- **Universal ingestion** вҖ” `.pdf`, `.md`/`.markdown`, `.txt` out of the box, behind a pluggable loader registry.
- **Image extraction** вҖ” figures embedded in PDFs are pulled out and shown in a gallery.
- **GenAIвҖ‘powered transformation** вҖ” the document is turned into structured study material using **Claude** (via the Anthropic API), with **OpenAI** as an alternative.
- **Reliable structured output** вҖ” uses the model's JSONвҖ‘schema structuredвҖ‘output mode, so the result always matches the expected shape.
- **Runs offline out of the box** вҖ” with **no API key**, a builtвҖ‘in heuristic "mock" provider generates a usable learning package so you can try the whole flow with zero setup.
- **Interactive UI** вҖ” tabbed mind map / summary / concepts / flashcards / quiz / images. The mind map is rendered with [markmap](https://markmap.js.org/), with a builtвҖ‘in fallback tree renderer when offline.
- **No build step on the frontend** вҖ” plain HTML/CSS/JS served by the backend.

1. **Ingestion** (`backend/ingestion/`) parses the upload into normalised text plus any extracted images.
2. **Pipeline** (`backend/pipeline/`) builds the prompt and hands the text + a JSON schema to the selected provider.
3. **GenAI provider** (`backend/genai/`) returns a learning package as schemaвҖ‘valid JSON.
4. **Frontend** (`frontend/`) renders it as an interactive workspace.

The generated learning package looks like this:

```jsonc
{
  "title": "...",
  "summary": "...",
  "mindmap": { "title": "...", "children": [ { "title": "...", "children": [...] } ] },
  "key_concepts": [ { "term": "...", "definition": "..." } ],
  "flashcards":   [ { "front": "...", "back": "..." } ],
  "quiz":         [ { "question": "...", "options": ["..."], "answer_index": 0, "explanation": "..." } ],
  "images":       [ { "id": "...", "url": "/media/<job>/img_000.png", "caption": "Figure 1 (page 2)" } ],
  "meta":         { "provider": "anthropic", "model": "claude-opus-4-8", "truncated": false, ... }
}
```

## III. Quickstart

### Option A вҖ” one command (recommended)

```bash
git clone https://github.com/ShenXiexs/visual-learn-everything.git
cd visual-learn-everything
./run.sh
```

Then open **http://127.0.0.1:8000** and drop in a file (or click **Try the
sample document**). With no API key set, it runs on the offline mock provider.

### Option B вҖ” manual

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn backend.main:app --reload
```

### Enable a real GenAI model

The offline mock is fine for a demo, but the quality jumps with a real model.
Set an API key and it's picked up automatically:

```bash
export ANTHROPIC_API_KEY=sk-ant-...      # uses Claude (claude-opus-4-8 by default)
# or
export OPENAI_API_KEY=sk-...             # uses OpenAI (gpt-4o-mini by default)
```

Or copy `.env.example` to `.env` and fill it in (autoвҖ‘loaded on startup).

## IV. Configuration

All settings are environment variables (see [`.env.example`](.env.example)):

| Variable               | Default            | Description                                                        |
| ---------------------- | ------------------ | ------------------------------------------------------------------ |
| `LLM_PROVIDER`         | auto               | `anthropic` \| `openai` \| `mock`. AutoвҖ‘detected from keys if unset. |
| `LLM_MODEL`            | `claude-opus-4-8`  | Model id (providerвҖ‘specific).                                      |
| `ANTHROPIC_API_KEY`    | вҖ”                  | Enables the Claude provider.                                       |
| `OPENAI_API_KEY`       | вҖ”                  | Enables the OpenAI provider.                                       |
| `VLE_MAX_INPUT_CHARS`  | `120000`           | Max characters of the document sent to the model.                 |
| `VLE_MAX_UPLOAD_BYTES` | `26214400` (25 MB) | Upload size cap.                                                   |
| `VLE_MAX_IMAGES`       | `40`               | Max images extracted per document.                                |
| `VLE_DATA_DIR`         | `./data`           | Where uploads + extracted images are stored.                      |

**Provider selection logic:** explicit `LLM_PROVIDER` wins; otherwise
`ANTHROPIC_API_KEY` вҶ’ Anthropic, then `OPENAI_API_KEY` вҶ’ OpenAI, else the
offline `mock` provider.

## V. API

The frontend is a thin client over two endpoints:

### `GET /api/health`
Returns the active provider, model, and limits.

### `POST /api/process`
`multipart/form-data` with a single `file` field. Returns the learningвҖ‘package
JSON shown above. Example:

```bash
curl -F "file=@samples/sample.md" http://127.0.0.1:8000/api/process
```

Interactive API docs are available at **http://127.0.0.1:8000/docs**.

## VII. Extending

**Add a new file type** вҖ” drop a loader in `backend/ingestion/` exposing
`load(path, media_dir) -> ExtractedDocument` and register its extension in
`registry.py`.

**Add a new GenAI provider** вҖ” implement `LLMProvider.generate_json()` in
`backend/genai/`, then wire it into `factory.py`.

**Change what gets generated** вҖ” edit the schema in `backend/genai/schema.py`,
the matching Pydantic models in `backend/models/schemas.py`, and the
instructions in `backend/genai/prompts.py`. The frontend reads each section by
name, so adding a field is mostly additive.

## IX. Notes & limitations

- This is a **demoвҖ‘grade** project meant to show the endвҖ‘toвҖ‘end concept; it
  processes requests synchronously and stores artifacts on local disk.
- Scanned/imageвҖ‘only PDFs won't yield text without OCR (not bundled).
- The `mock` provider is heuristic вҖ” use a real model for quality output.
